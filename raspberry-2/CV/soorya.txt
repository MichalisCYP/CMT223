💻 The Final Code
I have refined your code to include a check for the Haar Cascade files. On Raspberry Pi, the paths can sometimes differ, so I added a fallback check.
File Name: posture_monitor.py
Python
import cv2
import os

# 1. Path Setup - Raspberry Pi standard paths
face_path = '/usr/share/opencv4/haarcascades/haarcascade_frontalface_default.xml'
eye_path = '/usr/share/opencv4/haarcascades/haarcascade_eye.xml'

# Fallback to local cv2 data if system paths don't exist
if not os.path.exists(face_path):
    face_path = cv2.data.haarcascades + 'haarcascade_frontalface_default.xml'
    eye_path = cv2.data.haarcascades + 'haarcascade_eye.xml'

face_cascade = cv2.CascadeClassifier(face_path)
eye_cascade = cv2.CascadeClassifier(eye_path)

# 2. Camera Initialization
# If 0 doesn't work, try -1 or 1
cap = cv2.VideoCapture(0)

# Settings for Posture
STANDARD_Y = None  
CALIBRATION_THRESHOLD = 40 

print("Starting Monitor... Press 'q' to quit.")

while True:
    ret, img = cap.read()
    if not ret:
        print("Failed to grab frame")
        break
    
    img = cv2.flip(img, 1) 
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    
    # Detect faces
    faces = face_cascade.detectMultiScale(gray, 1.3, 5)

    status = "Looking Away!"
    color = (0, 0, 255) # Red

    for (x, y, w, h) in faces:
        # Draw face box (Cyan)
        cv2.rectangle(img, (x, y), (x+w, y+h), (255, 255, 0), 2)
        
        # Calibrate posture on first sight: 
        # We assume the first time you are seen, you are sitting straight.
        if STANDARD_Y is None: 
            STANDARD_Y = y
            print(f"Calibrated! Standard Y level: {STANDARD_Y}")
        
        # Check Posture (If Y increases, your head is lower = slouching)
        if y > STANDARD_Y + CALIBRATION_THRESHOLD:
            cv2.putText(img, "FIX POSTURE!", (10, 110), 
                        cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 0, 255), 3)

        # ROI (Region of Interest) for eyes
        roi_gray = gray[y:y+h, x:x+w]
        eyes = eye_cascade.detectMultiScale(roi_gray, 1.1, 10)
        
        if len(eyes) >= 2:
            status = "Focused"
            color = (0, 255, 0) # Green
            for (ex, ey, ew, eh) in eyes:
                cv2.rectangle(img, (x+ex, y+ey), (x+ex+ew, y+ey+eh), (0, 255, 0), 2)
        else:
            status = "Eyes Not Detected"

    # UI Overlay
    cv2.putText(img, f"Status: {status}", (10, 50), 
                cv2.FONT_HERSHEY_SIMPLEX, 1, color, 2)
    
    cv2.imshow('Posture & Focus Monitor', img)

    # Press 'q' to exit or 'r' to re-calibrate posture
    key = cv2.waitKey(1) & 0xFF
    if key == ord('q'):
        break
    elif key == ord('r'):
        STANDARD_Y = None
        print("Re-calibrating...")

cap.release()
cv2.destroyAllWindows()
🔍 Why this fixed it
The NumPy Crash: You had NumPy 2.0.2. OpenCV (and Mediapipe) were built against NumPy 1.x. When you tried to run them, the "C-API" versions didn't match. Downgrading to numpy<2 (specifically 1.26.4) is the industry-standard fix right now until those libraries update.
The pip Missing Error: When you created the venv with --without-pip, you deleted your ability to install things inside the environment. Re-running the environment creation properly (or using the curl method you did) restored the local pip binary.
Display Environment: Using export DISPLAY=:0 tells the Raspberry Pi to send the camera window to the actual monitor connected to the Pi (important if you are using SSH).
AttributeError Fix: Using cv2.data.haarcascades ensures that even if you don't know exactly where the XML files are, Python will find them inside the OpenCV installation folder.
